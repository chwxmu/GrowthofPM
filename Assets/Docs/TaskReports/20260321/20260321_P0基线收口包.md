# P0基线收口包

## 任务背景

- 本次任务仅执行路线图中的 P0 基线收口包，不扩展到 Phase 3/4/5 的新增玩法开发。
- 目标是在保护现有数据驱动周流程骨架的前提下，收口四个会持续影响后续开发与论文数据质量的阻塞问题：
  - 决策后 `aiTrustRecords` 未即时保存
  - Quiz 奖励与 Schedule 可分配精力闭环不一致
  - P2 `eventFlags` 与 P3 `AITrustRecord` 数据契约未冻结
  - 动态生成 TMP 中文字体显式绑定方式不稳定

## 核心逻辑

### 1. 决策日志即时保存

- 在 `GameManager.RecordAIDecision()` 中补入即时 `SaveProgress()`，让 `aiTrustRecords` 在单次决策完成后立即落盘，而不是等到周结算。
- 同时为 `AITrustRecord` 补齐后续统计需要的基础字段：
  - `projectNumber`
  - `weekNumber`
  - `aiQuality`
- `StoryManager.OnDecisionOptionSelected()` 现在会把当前决策的 `aiQuality` 一并传给 `GameManager.RecordAIDecision()`。

### 2. Quiz 与 Schedule 精力闭环统一

- `StoryManager.ShowSchedulePanel()` 改为优先使用 `CurrentPlayerData.energy` 作为当前周可分配精力，而不再强制写死为 `300`。
- `SchedulePanel` 新增缓存同步逻辑，在从 Quiz 返回 Schedule 时，会保留已选任务，同时按“当前实时精力 - 已选任务消耗”重算剩余精力。
- `StoryManager.OnScheduleComplete()` 改为按当前实时精力结算剩余值，避免 Quiz 奖励在确认日程后被重置掉。

### 3. 冻结 P2 / P3 数据契约

- 由于当前存档仍基于 `JsonUtility`，没有采用 `Dictionary<string, bool>`，而是为 `PlayerData` 增加 `List<EventFlagRecord> eventFlags`，确保可稳定序列化。
- `EventFlagRecord` 当前字段为：
  - `flagId`
  - `projectNumber`
  - `value`
- `GameManager` 新增 `SetEventFlag()` 与 `TryGetEventFlag()`，`StoryManager` 的占位旗标逻辑改为走可持久化的正式契约。

### 4. 修正动态 TMP 中文字体显式绑定

- 为 `DialoguePanel`、`EndingPanel`、`TransitionPanel` 补齐 `_preferredChineseFont` 序列化字段，并统一通过 `OnValidate()` 自动指向 `Assets/Fonts/SIMSUN SDF.asset`。
- `DecisionPanel`、`SchedulePanel`、`QuizPanel` 继续使用 `_preferredChineseFont`，但移除了依赖 `Resources.FindObjectsOfTypeAll`、默认字体或场景中任意现有字体的非稳定回退方式。
- 保存 `GameScene` 后，当前主流程核心面板都已写入指定中文字体引用，后续运行时动态创建的 TMP 文本会优先显式绑定该字体资源。
